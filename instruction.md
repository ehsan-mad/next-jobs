# UI Development Instructions & Guidelines

This document outlines the strict rules and guidelines for building the UI from the provided Figma file (View-Only Access).

## 🛑 Core Rules
1. **Fonts:** Ask for the exact font used in the Figma file before implementing any text styles.
2. **Colors:** Use the exact colors specified in the Figma design. Always confirm hex codes/RGBA values.
3. **Styling:** The styling of the UI must precisely match the Figma design.
4. **Pacing:** DO NOT build the entire UI in a single generation. Build strictly **component by component**.
5. **Validation:** After a component is finished, pause and ask for a review to re-evaluate the UI before moving forward.
6. **Spacing:** Always ask for exact margin and padding values if they are not abundantly clear.
7. **Assets:** Ask for pictures, icons, or SVG exports when needed for an exact UI match.
8. **Responsiveness:** Every component and layout must be responsive (mobile, tablet, desktop).
9. **Visual Context:** Screenshots from Figma will be provided to help understand and build the design. Ask for them!
10. **Environment:** Keep building optimized for local development.
11. **Animation:** Provide minimal, tasteful animations (e.g., hover states, smooth transitions) where possible and appropriate.
12. **Backend Integration:** Backend integration will ONLY start after the *entire* Figma UI design is fully finished and approved.
13. **Clarification:** Ask for any needed information before starting to build anything. Never start without a full idea or if there is any confusion.
14. **Data:** A JSON documentation of the backend will be provided when the frontend UI is completed.
